第一个部署在hexo的博客

2019.05.26 lxlandzyq  热度 ℃

hexo的问题

---系统：ubuntu


首先说一下自己在hexo中遇到的问题，本人是在ubuntu上部署的，按照官网的需求，需要安装git和nodejs。由于本人已经有git，所以就只需要安装nodejs
安装nodejs时候遇到的问题，因为偷懒，所以是采用命令来安装的
代码： sudo apt-get install nodejs
sudo apt-get install npm

在这里就需要注意，安装的版本是4.多的版本
导致我在后来一直hexo的时候都会报错，在安装hexo的时候以为一切顺畅，nodejs版本太低，导致hexo命令一致在报错

所以只能重新安装nodejs

正式过程

安装nodejs sudo apt-get install npmwget https://nodejs.org/dist/v10.9.0/node-v10.9.0-linux-x64.tar.xz tar xf node-v10.9.0-linux-x64.tar.xz cd node-v10.9.0-linux-x64/

在显示版本号的时候则要修改其配置
vim /etc/profile //修改环境变量，用sudo打开

修改配置

#set for nodejs
export NODE_HOME=/usr/local/node/0.10.24 //在其相应的路劲下进行选择
export PATH=$NODE_HOME/bin:$PATH

source /etc/profile //验证是否配置成功
node -v //显示版本号的话证明配置成功 接下来重启的就可以显示了

按照官网给的教程

npm install -g hexo-cli

然后新建一个文件夹
mkdir blog
hexo init blog
cd blog
npm install
hexo s //部署服务
输入网址：localhost:4000
终于打开了！！！！！

部署到github上
首先修改相应的deploy里面的内容
修改内容为
deploy:
type: git
repo: https://github.com/lxlandzyq/lxlandzyq.github.io.git

//其中repo里面的内容可以是github上面的git也可以是https

安装git部署的插件
npm install hexo-deployer-git –save

hexo clean
hexo g //生成静态文件
hexo d //部署网站

输入xxxx.github.io就会显示由hexo部署的blog

写文章
hexo n name //name为blog名字
然后它会创建一个文件给你，在指定的文件夹里面可以看到这个文件
然后进行编辑

在撰写的时候手贱想是一下debug 就用来命令 hexo s –debug
但是没有看到什么效果，暂时还不懂用
hexo g
hexo d
然后一直不显示在页面上
最后把当前文件夹里面把debug这个文件删除了
hexo clean
hexo g
hexo d
就显示在界面上了
