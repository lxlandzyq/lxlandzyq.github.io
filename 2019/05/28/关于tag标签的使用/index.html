关于git中的tag的介绍

在git中的tag可以控制版本的信息，并获取需要制定的版本的内容，这样子有利于当遇到错误的时候也可以回到之前的版本进行相应的修改。

tags在github中的位置

如何使用git中的tags

1.首先我们可以先将工程git 下来

2.然后可以对次工程就行做相应的修改，修改完之后就就可以add

3.添加注释

4.提交到远程仓库中

注释：二三步可以做好相应的修改再git push 提交上去

5.为刚刚提交的项目添加tags

git tag为打标签的命令，-a 为添加标签，其中v1.01为要给这个工程添加的标签名字 -m 后面的为这个版本的注释

6.将标签提交到github上

结束！

扩展：
1.查看tags有几个版本
git tag //将会列出这个工程所有的版本

2.查看tags的信息
git checkout +版本标签 //则会显示这个标签的所有内容

3.显示所有标签并显示其注释
git tag -l -n //显示所有的标签，并显示其标签的注释
