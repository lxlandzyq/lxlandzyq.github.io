若依前后端项目分离部署

2020.07.08 lxlandzyq  热度 ℃

若依项目

若依有前后端结合和前后端分离两个项目，该项目也是Spring boot项目，通过此可以快速的了解框架，项目地址：https://gitee.com/y_project/RuoYi-Vue
通过git将其下载到本地。项目结构如下：

1:为后端内容
2:为前端内容
在此项目中，宿主地址为：172.20.10.2
虚拟机地址为：172.20.10.5，该ip地址存入数据库和项目

前期准备

安装软件

在上篇已经设置了Linux多节的搭建，在此处进行环境的搭建，为项目设置其环境。百度连接如下：

数据导入

通过navicat对数据的导入，连接172.20.10.5IP地址等数据库，创建ry-vue数据库，再execute sql file文件，选择数据库文件导入

修改配置

因为项目的环境都是在虚拟机上即172.20.10.5这个机子中，所以要在本地项目中修改其数据库和redis等具体ip地址。



修改数据库指定等ip地址，以及logs的位置，查看其日志

项目移动

压缩ruoyi-vue整个项目，通过transmit软件将压缩包移动到172.20.10.5这个机子里。
压缩的目的是为了防止一些隐藏文件夹没有移动过来。

前端打包

在172.20.10.5这台机子上，将从宿主机子移过来的压缩包解压

进入前端文件夹
将其添加依赖

利用cnpm将其打包

前端打包成功后会在当前目录下出现一个dist的文件夹

后端运行

进入后端文件夹

将其打包

打包成功会出现一个target的文件夹
target文件夹中ruoyi.jar包放在/home/lxl/workplace/ruoyi进行备份

后端部署

启动后端，让其在后台一直运行

进到后端目录下

在启动台上启动，如果启动失败可以在当前目录下有个nohup.out查看日志，其出现对错误
在本次出现对错误为：

通过查找发现是打包的Java环境要与工程的Java的环境一致，即为

Java 版本为1.8 原来centos系统自带java，其版本为1.7所以打出的jar包就无法使用，最后用1.8版本对其打包完成操作。

根据上一张图片可以知道，前端跳转的目录为/prod-api，再对
/usr/local/nginx/conf目录下的nginx.conf进行添加后端内容

/prod-api/即为跳转的路径，其中proxy_set_header为固定内容
proxy_pass 为后端跳转的路径
其中172.20.10.5:8080 为后端存放的ip地址

最后重启nginx

最后再输入地址
172.20.10.5
即可查看前后端结合的项目来

注意

在启动过程可能会出现端口占用的情况，可以将其kill掉再重新启动


查看其80端口号被占用的情况
如果被占用无法启动，将其kill掉
