git添加tag标签，使其可以查看每一个版本的源码

对每一个版本的测试，给每一个版本添加标签
git tag -a v1.01 -m "描述版本信息"
命令分析：
git tag为打标签的命令，-a 为添加标签，其后输入的为其标签的版本号 -m进行描述信息 git tag //将会列出所有版本号
git checkout +版本号 //当前的文件夹将会变成此版本的信息
当在git tag -a v1.01 -m ""中只为本地的版本信息，要想同步到github上面则必须要同时提交tag
命令为:git push --tags
测试一些数据，对应tag版本
再进行测试数据；
进行对数据的测试之后发现，再进行提交的时候先提交git push将本地仓库提交到github上面
然后再 git tag -a v -m "" //添加标签
添加标签之后，然后再进行提交标签 git push --tags //此时就是对应的tag对应其版本好
